# Weekly Map

## Week 1: Introduction to Coding LLMs and AI Development

Core ideas:
- What an LLM is in practical engineering terms
- How prompting changes the software workflow
- Why prompt design is a lever, not a cosmetic trick

Practical emphasis:
- K-shot prompting
- Chain-of-thought and self-consistency
- Tool calling
- RAG and reflexion

Interpretation:
The course starts by making students operationally literate with model behavior before introducing heavier systems concepts.

## Week 2: The Anatomy of Coding Agents

Core ideas:
- Agent architecture and components
- Tool use and function calling
- MCP (Model Context Protocol)

Practical emphasis:
- Build an LLM-powered extractor
- Add tests and endpoint changes
- Start using an AI IDE in a codebase with state and structure

Interpretation:
This is the transition from prompting isolated models to building systems that can act through interfaces.

## Week 3: The AI IDE

Core ideas:
- Context management and code understanding
- PRDs/specs for agents
- IDE integrations and extensions

Practical emphasis:
- Build a custom MCP server
- Learn that context engineering is a first-class software task

Interpretation:
This week reframes the editor as a structured orchestration surface, not just a text buffer.

## Week 4: Coding Agent Patterns

Core ideas:
- Managing autonomy levels
- Human-agent collaboration patterns

Practical emphasis:
- Build automations inside Claude Code
- Use slash commands, repo guidance files, subagents, and MCP integrations

Interpretation:
This is where the course starts teaching operational governance, not just capability.

## Week 5: The Modern Terminal

Core ideas:
- AI-enhanced command-line interfaces
- Terminal automation and scripting

Practical emphasis:
- Agentic development with Warp
- Multi-agent workflows in terminal-native environments

Interpretation:
The terminal is treated as an execution surface for agents, not just a shell.

## Week 6: AI Testing and Security

Core ideas:
- Secure vibe coding
- Vulnerability detection history
- AI-generated test suites

Practical emphasis:
- Use Semgrep
- Fix concrete issues
- Explain mitigations precisely

Interpretation:
This week is a corrective against naive tool enthusiasm. AI output without security discipline is treated as operationally incomplete.

## Week 7: Modern Software Support

Core ideas:
- Trust in AI code systems
- Debugging and diagnostics
- Intelligent documentation generation

Practical emphasis:
- Compare manual review with AI-assisted review
- Use PRs, branches, and review workflows explicitly

Interpretation:
The course acknowledges that generating code is only one slice of the lifecycle; support, review, and maintainability are equally important.

## Week 8: Automated UI and App Building

Core ideas:
- Design and frontend for everyone
- Rapid UI/UX prototyping and iteration

Practical emphasis:
- Build the same app in three stacks
- Include at least one AI app-generation platform
- Include at least one non-JavaScript stack

Interpretation:
This week tests how far AI acceleration can stretch across stack boundaries.

## Week 9: Agents Post-Deployment

Core ideas:
- Monitoring and observability for AI systems
- Automated incident response
- Triaging and debugging

Practical emphasis:
- Bring SRE concepts into AI-native engineering
- Treat operations as part of the agent lifecycle

Interpretation:
This is one of the strongest weeks in the course because it connects coding agents to production reality.

## Week 10: What's Next for AI Software Engineering

Core ideas:
- Future software roles
- Emerging paradigms in AI coding
- Trend forecasting

Interpretation:
The final week forces students to convert tool familiarity into a view of how engineering organizations may change.

## Syllabus Compression

If the 10-week course is compressed into four themes, it becomes:

1. Model literacy and prompting
2. Agent architecture and developer workflows
3. Verification, security, and review
4. Deployment, operations, and role evolution
