# 首发文案包

## GitHub 仓库简介

`在那些你最不甘心的节点里，亲手改写结局。一个可直接试玩的互动叙事游戏 MVP。`

## GitHub 首发贴文

```text
我在做一个互动叙事游戏 MVP，名字叫《改命剧场》。

它的核心不是“看剧情”，而是“进入一个大家最不甘心的节点，亲手改写结局”。

当前公开的三支 Demo 是：
1. 《麦城前夜》
2. 《鸿门宴前席》
3. 《风雪山神庙前夕》

前者在问：活下来更重要，还是传奇更重要？
后者在问：先杀最大的对手，还是先稳住天下人心？
第三支在问：先活下来更重要，还是先把仇翻成一份说得清的证据更重要？

而且每支剧本里都埋了更极端的隐藏结局，结算页会直接告诉你这轮路线到底是“最燃”“最阴”“最反意难平”还是“最气人”。

仓库和试玩页都已经放出来了，欢迎试玩和提 issue：
```

### 单剧本切片版

```text
如果麦城前夜能重来一次，你会怎么选？

我做了一个可试玩的互动叙事 Demo《改命剧场：麦城前夜》。
这次不是“看关羽怎么走向结局”，而是你亲自接管那一夜的决策。

你可以试：
1. 保人
2. 保名

4. 赌突围

我想验证的不是“历史能不能改”，而是大家到底更在乎：
```

## 小红书文案

### 单剧本版

```text
我最近在做一个有点上头的互动叙事游戏原型。

它叫《改命剧场》，玩法很简单但很戳我：
不是看角色走剧情，而是直接进入“最意难平的节点”，自己做选择，看看结局会不会被改掉。

我先做了《麦城前夜》这个 Demo。
如果那一夜由你接管决策，你会：
- 放下面子先救人
- 继续守住传奇姿态
- 还是赌最后一次突围？

最有意思的是，玩完以后会发现：
有些结局更燃，但不一定更值；
有些结局不体面，却更像活人真的会做的选择。
```

```text
我最近把一个互动叙事游戏原型继续做下去了，名字叫《改命剧场》。

- 麦城前夜

- 风雪山神庙前夕

我最上头的一点是：
同一个节点，真的能走出完全不同的命。

而且每一支里面我还埋了更极端的隐藏结局。
不是简单多一个分支，而是会让你突然意识到：
```

## 30 秒视频脚本

### 镜头 1

字幕：
`如果麦城前夜能重来一次呢？`

旁白：
`如果你能在麦城前夜接管决策，你会怎么选？`

### 镜头 2

展示 Demo 首屏和三个切入节点

字幕：
`失荆州消息初到 / 麦城前夜 / 突围前一刻`

旁白：
`我做了一个互动叙事游戏原型，把那些最让人意难平的节点，做成可以亲自改命的选择。`

### 镜头 3

快速切两个不同选择

字幕：
`保人？保名？赌援军？`

旁白：
`你每做一次选择，角色命运、局势和结局评分都会变。`

### 镜头 4

展示两个差异很大的结局

字幕：
`更燃，不一定更值。更难看，不一定更错。`

旁白：
`有的结局更燃，但几乎走回原本的败局；有的结局不够传奇，却真的让更多人活下来。`

### 镜头 5

展示结局页和分享文案按钮

字幕：
`你会怎么选？`

旁白：
`如果你来改这场戏，你会选哪条路？`
