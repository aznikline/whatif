# 改命剧场 MVP 路线

## 先说结论

你现在最需要的不是 Steam 首发，也不是“大而全的 AI 剧本平台”。

你最需要的是一个能在 7 到 14 天内拿出去测试、能让人一眼理解、能产生传播截图和评论区争论的最小可玩版本。

当前最优路线：

1. `Web/H5 可玩原型` 作为主载体
2. `B 站 / 小红书 / 抖音` 作为传播渠道
3. `Steam Demo` 放到第二阶段

## 为什么不是 Steam 首发

Steam 适合第二阶段，不适合你当前阶段，原因很直接：

1. 上线链路更长，愿望单积累慢
2. 你现在最需要的是高频迭代，不是平台流程
3. 你的卖点是“意难平改命”，它更适合短视频和截图传播
4. 你现在还在验证：用户到底是为“改命”买单，还是为“IP 代入”买单

Steam 适合在你已经有：

1. 一个反复打磨过的高留存剧本
2. 一批传播素材
3. 一个明确的商业包装页

之后再上。

## 为什么也不建议先做纯 B 站互动视频

B 站互动视频能做传播样片，但不适合作为主 MVP。

问题有三个：

1. AI NPC 的持续记忆和成长很难真正表现
2. 更新内容要重新剪辑和上传，迭代成本高
3. 玩家会把它当“分支视频”而不是“活的改命系统”

更好的做法：

1. `可玩 Web 原型` 做核心体验
2. `B 站互动视频` 做导流样片和话题爆点

也就是：

`互动视频负责吸睛，Web 原型负责证明它真的能玩。`

## 当前阶段的核心目标

不是证明“什么都能做”。

而是证明这三件事：

1. 用户会因为“改写意难平”点进来
2. 用户会因为 NPC 真有反应而继续玩
3. 用户会愿意把自己的改命结果发出去

## MVP 一期必须包含的功能

只做下面这些：

1. 选择剧本
2. 从关键节点切入
3. 3 到 5 个关键选择
4. 2 到 4 个核心 NPC
5. NPC 关系状态变化
6. 结局判定
7. 结算评分
8. 可分享的结局摘要

## MVP 一期不要做的功能

先克制住，不然会死在复杂度里：

1. 海量剧本库
2. 真的“任何电影、小说、电视剧都能玩”
3. 完整开放世界
4. 多人联机共演
5. 复杂实时语音
6. 完整 UGC 剧本市场

## 公开发布时的版权策略

这个点不能回避。

如果你公开发布、商业化或大规模传播，直接使用知名影视小说角色与剧情，风险很高。

所以 MVP 建议分成两层：

1. `公开版`
   - 用原创剧本
   - 但保留强烈的“剧感”
   - 用“误会、诀别、救人、背叛、权谋”这些通用情感结构验证玩法
2. `封闭测试版`
   - 给种子用户体验“已知作品改命”
   - 不公开大规模传播

也就是：

`先验证玩法价值，再解决 IP 路线。`

## 最适合破圈的第一支剧本

第一支剧本不要做太大，建议满足这几个条件：

1. 有强烈遗憾感
2. 有一个明确“如果当时换个选择”的节点
3. NPC 关系足够纠缠
4. 结局能分出明显层级
5. 截图就能看懂戏剧冲突

建议第一支剧本结构：

- 类型：古风权谋爱情
- 关键词：误会、护送、密信、牺牲、回城、错认
- 角色：
  - 玩家：局中人
  - 谋士：知道更多真相，但不肯一次说透
  - 执行者：负责推进现实局面，但会和玩家起冲突
  - 目标角色：玩家最想救、最想留住的人

## 推荐的最小技术形态

最省钱、最快速、最好迭代的组合：

1. `静态 Web 页面`
2. `前端本地状态机`
3. `少量伪动态 NPC 成长`
4. `结局卡片 + 分享文案`

先不要把 LLM 放到核心路径里。

原因很现实：

1. 成本不稳定
2. 响应速度不稳定
3. 你现在需要先证明“玩法成立”

第一版可以是：

- 结构化剧情
- 准动态文本
- 状态驱动 NPC 反应

等用户喜欢这个核心体验，再把 Agent 真接上去。

## 两周执行计划

### 第 1 周

1. 做出单剧本可玩 Demo
2. 完成 3 个切入节点
3. 完成 4 个结局
4. 做出结算页和分享页

### 第 2 周

1. 打磨文案和视觉风格
2. 做 3 条短视频切片
3. 做一支 B 站互动视频样片导流
4. 小范围投放测试

## 传播内容怎么做

传播重点不是“AI 很强”，而是“你终于能改那个结局了”。

可直接用的传播标题方向：

1. 如果那一夜你拦住了他，结局会改写吗？
2. 我把最意难平的一场诀别改了，结果更刀了
3. 你以为救下他就够了？其实真正的代价在后面
4. 这个游戏专门让你重写那些不甘心的剧情

## 成功标准

MVP 不看 DAU，不看大而空的“行业想象力”。

先看：

1. 首轮游玩完成率
2. 二周目占比
3. 分享率
4. 评论区争论强度
5. 用户会不会主动说“我要试另一个选项”

## 下一步最合理的产品路线

### 阶段 A

- 一个原创剧本
- 一个网页原型
- 一套传播素材

### 阶段 B

- 3 个风格不同的剧本
- 增强 NPC 记忆
- 增加“从任意节点切入”

### 阶段 C

- 接入真正的 Agent NPC
- 开始封闭测试 IP 改命玩法
- 准备 Steam Demo 页面

## 现在最该做的事

立刻做出一个玩家能玩完、能截图、能吵起来的原型。

不是做平台。
不是做宇宙。
不是做全世界的剧本。

先做一个能让人说出这句话的东西：

`“我靠，我真的把那场戏改了。”`
